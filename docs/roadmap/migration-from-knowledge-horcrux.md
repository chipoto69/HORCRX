# Migration from ATLAS knowledge-horcrux

ATLAS → CORPUS → HORCRX.

Source prototype: `~/.claude-worktrees/ATLAS/knowledge-horcrux/atlas/`. The prototype is read-only. This table maps each root module to the future HORCRX destination package or explicitly parks it out of scope.

| ATLAS module | Current role | HORCRX destination | Migration posture |
|---|---|---|---|
| `__init__.py` | package marker | `packages/horcrx-core/src/` | recreate, do not copy blindly |
| `AGENTS.md` | local agent instructions | `docs/roadmap/migration-from-knowledge-horcrux.md` references only | read-only source, not runtime package |
| `agent.py` | monolithic agent entrypoint | `packages/horcrx-cli/` and `packages/horcrx-runtime-adapters/` | split by responsibility in Phase 1+ |
| `cli.py` | broad CLI surface | `packages/horcrx-cli/` | port only vessel-relevant commands |
| `agents/` | AgentCard, onboarding, wallet, reputation, horcrux prototype surfaces | `packages/agent-card/`, `packages/horcrx-cli/`, `packages/reputation-adapters/` | selectively port schemas and adapters |
| `api/` | FastAPI routers and middleware | `apps/api/` | future service layer, not foundation code |
| `billing/` | x402, chain verifiers, subscriptions, escrow, settlement | `packages/x402-gateway/`, `packages/chain-adapters/`, `packages/payment-ledger/` | port x402 and verifier seams first |
| `billing_cli/` | wallet, budget, payments CLI | `packages/horcrx-cli/` | merge only user-facing payment verbs |
| `bridges/` | OpenClaw, ElizaOS, Moltbook, identity bridges | `packages/runtime-bridges/` | park until post-CLI runtime bridge phase |
| `cli/` | small CLI modules | `packages/horcrx-cli/` | merge into command modules |
| `core/` | DB, registry, memory, services, protocols | `packages/registry-indexer/`, `packages/horcrx-core/`, `packages/memory-adapters/` | split into core protocol vs indexer concerns |
| `data/` | local SQLite data | out of repo fixtures only | never port live database state |
| `economy/` | economic configuration | `packages/marketplace-policy/` | port constants only after Phase 6 decisions |
| `knowledge/` | knowledge commerce and x402 gate | `packages/ip-preservation/` and `packages/marketplace-policy/` | reuse concepts, not direct state |
| `lib/` | shared helpers such as Supabase | `packages/platform-adapters/` | defer until service implementation needs it |
| `marketplace/` | marketplace models, service v1/v2, connectors | `apps/marketplace-api/`, `packages/marketplace-core/`, `packages/connectors/` | port listing and trust primitives |
| `mcp/` | MCP server/auth | `packages/horcrx-marketplace-mcp/` | Phase 4 primary source |
| `social/` | feed, reactions, autoposter | `packages/go-to-market-connectors/` | Phase 8 only |
| `swarms/` | coordinator and models | `packages/orbel-pack-runtime/` | use as reference for multi-agent routing |
| `tasks/` | task models and service | `packages/task-ledger/` | defer until agent marketplace operations need it |

## Porting rules

1. Preserve HORCRX vessel and protocol specs as the authority; ATLAS code is prior art.
2. Never port live databases, secrets, caches, or generated bytecode.
3. Keep Base and Solana adapters symmetric at the protocol boundary.
4. Preserve AgentCard compatibility rather than inventing a competing card model.
5. Every ported module gets tests and a migration note that cites the source path.
